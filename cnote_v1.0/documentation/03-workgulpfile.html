Working with Gulp

Gulp is a JavaScript task runner. It helps with performing repetitive tasks like minification, compilation, etc. This is possible thanks to extensive Gulp Plugins library.

Gulp tasks are already set up for you. They are stored inside cnote/gulpfile.js.

gulp start this main command will fire default gulp task which includes: launching BrowserSync, vendor files concatination, javascript minification, sass compilation and lanching watch task. BrowserSync will create dev server and sync your browser with your code editor. Note: to use other separate tasks explained below you will need to open other Terminal window and leave this one with running server intact.

You can also run them separately, by typing the following tasks in your command line :

  • gulp html - compiles all .html template files from src/ into dist/ folder.
  • gulp main-js - compiles the js code from src/ into dist/ folder.
  • gulp main-scss and gulp vendor-scss - compiles .scss files into non-minified CSS from src/ to dist/ folder.
  • gulp vendor-js copy all js libs / plugins you need from /node_modules into /dist/assets/vendor/js/ folder.

    Check the gulpfile.js ( vendor-js task ) if you want to add / change something.

  • gulp cssminify - minify all css files from dist/ .
  • gulp jsminify - minify / uglify all js files from dist/
  • gulp minify - you can run this task instead of the two before. (cssminify and jsminify). It will be the same result.
  • gulp watch - launches watch task. It will watch changes of your .scss / .js / .html files and automatically compile them into .css / .js / .html from dist/ folder.
  • gulp fonts - compile necessary fonts from node-modules/.
  • gulp images - copy all images from src/into dist/ .

Note: If you open a page in Firefox on local system like " file:///C:/cnote/your-path-file.html/ " the fonts will not be displayed !
Try Google Chrome instead or gulp start to spin up the server locally

Firefox has security restrictions on accessing files outside the directory the HTML document lives in when loading files from the local file system. Fonts are one of the resources impacted by this.
